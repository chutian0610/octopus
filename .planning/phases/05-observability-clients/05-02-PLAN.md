# JDBC Driver Architecture
JDBC Type 4 drivers implement:
- `java.sql.Driver` — entry point, parsed JDBC URL
- `java.sql.Connection` — handles a connection to database
- `java.sql.Statement` — executes SQL statements
- `java.sql.PreparedStatement` — pre-compiled SQL
- `java.sql.ResultSet` — tabular results

In Rust, use `jdbc` crate for type bindings and implement the traits.
Coordinator HTTP API on port 50051 accepts query submission.
</context>

<task type="auto">
  <name>Task 2: Implement Connection, Statement, ResultSet</name>
  <files>octopus-jdbc/src/connection.rs, octopus-jdbc/src/statement.rs, octopus-jdbc/src/result_set.rs</files>
  <action>
Implement `OctopusConnection`:
- Stores host:port and HTTP client (reqwest)
- `createStatement()` returns OctopusStatement
- `close()` drops connection
- `isClosed()` tracks state

Implement `OctopusStatement`:
- `executeQuery(sql)` -> ResultSet
- Calls coordinator HTTP endpoint: POST /query/submit
- Polls /query/state/{id} until completed
- Returns results via `OctopusResultSet`

Implement `OctopusResultSet`:
- Holds query results (Vec of rows, schema)
- `next()` iterates rows
- `getString(col)`, `getInt(col)`, `getDouble(col)`, `getTimestamp(col)` fetch values
- `getMetaData()` returns column info

Use JNI for Java objects. Each Rust struct holds jclass (class reference) and jobject (instance).
  </action>
  <verify>grep -c "impl.*Connection\|impl.*Statement\|impl.*ResultSet" octopus-jdbc/src/*.rs</verify>
  <done>Connection, Statement, ResultSet JDBC interfaces implemented</done>
</task>
