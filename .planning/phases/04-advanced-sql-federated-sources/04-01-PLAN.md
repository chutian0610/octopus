## Context Decisions (D-01 through D-04)

### D-01: Connection Pool Per Worker
**Decision:** Maintain a connection pool per worker for PostgreSQL/MySQL connections.

### D-02: Window Functions — Auto Mode
**Decision:** Let DataFusion decide execution mode (streaming vs bounded) based on partition size.

### D-03: Function Coverage — Built-in + Custom
**Decision:** Use DataFusion built-in functions for standard cases; add Octopus-specific custom implementations for performance-critical functions.

### D-04: Type Mapping — Adapter Pattern
**Decision:** Create a unified type abstraction with per-database adapters for PostgreSQL and MySQL.
</context>

2. **TypeAdapter trait** — Abstracts database-specific type mapping:
```rust
pub trait TypeAdapter: Send + Sync {
    fn database_type(&self) -> DatabaseType;
    fn to_arrow_type(&self, sql_type: &str) -> Result<DataType>;
    fn from_arrow_type(&self, arrow_type: &DataType) -> Result<String>;
}
```

3. **ConnectionPool trait** — Abstracts pool operations:
```rust
pub trait ConnectionPool: Send + Sync {
    type Connection;
    async fn get(&self) -> Result<Self::Connection>;
    async fn release(&self, conn: Self::Connection) -> Result<()>;
    fn stats(&self) -> PoolStats;
}
```

4. **FederatedConnector trait** — Main interface for federated datasources:
```rust
pub trait FederatedConnector: Send + Sync {
    fn database_type(&self) -> DatabaseType;
    fn type_adapter(&self) -> Arc<dyn TypeAdapter>;
    fn connection_pool(&self) -> Arc<dyn ConnectionPool>;
    fn get_schema(&self, table: &str) -> Result<Schema>;
    fn execute_query(&self, sql: &str) -> Result<RecordBatchStream>;
}
```

5. **PoolStats struct** — For connection pool monitoring.

Use Arc<dyn Trait> for dynamic dispatch. Import from datafusion for DataType and Schema.
